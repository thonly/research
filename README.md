## Why defensive publication, not patent

Each publication in this repository represents original mechanism design that could, in principle, be patented. The author has chosen defensive publication for converging reasons:

1. **Mission alignment.** The work is dharma-grounded; the higher form of giving is giving without expectation of return. Patenting dharmic infrastructure is internally incoherent.
2. **Equal protective effect.** Defensive publication and patent provide the same protection against later patent filings by other parties: both establish prior art that defeats subsequent claims. Defensive publication is faster, cheaper, and permanent (patents expire in 20 years; prior art does not).
3. **Adoption-friendly.** Patents block adoption; defensive publications enable it. Infrastructure-class innovations succeed as commons (TCP/IP, HTTP, DNS were defensively published; the result is the entire internet).
4. **Right signal.** Mission-driven publishing signals the right values to the alliance the work is intended to serve.

Each publication explicitly states (a) public-domain dedication under CC0, (b) patent non-assertion by the author and HeartBank®, and (c) prior-art declaration as of the publication date.

## Document structural template

Every publication in this repository follows the same skeleton, established at [`thonly.org/research/template`](https://thonly.org/research/template):

1. **Metadata header** — title, author, date, canonical URL, license, document SHA-256
2. **Preamble** — dharma framing as a *dāna* offering
3. **Prior-art and non-assertion statement** — three components: CC0 dedication, patent non-assertion, prior-art declaration with explicit date
4. **Abstract** — self-contained summary, 200–400 words
5. **Numbered body sections** — typically 10–13: Introduction; Background and Prior Art; the novel contribution; System Architecture; Reference Implementation; Edge Cases; Use Cases; Limitations; etc.
6. **Cross-venue references** — canonical, institutional mirror, GitHub, arXiv, IP.com, Internet Archive (web.archive.org / archive.today / perma.cc)
7. **Acknowledgments**
8. **Citations** — numbered list, dharma sources cited per Pāli Text Society conventions

Anyone is welcome to use this template structure as the skeleton of their own defensive publications. The template itself is dedicated to the public domain.
